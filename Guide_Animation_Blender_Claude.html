Concepts clés de l'animation

Keyframe

Point clé dans la timeline qui enregistre la valeur d'une propriété à un instant donné. Raccourci : I pour insérer.

Timeline

Ligne temporelle où sont placés les keyframes. Unité = frame (image). La barre de lecture avance frame par frame.

Frame Rate (FPS)

Nombre d'images/seconde. 24 fps (cinéma), 30 fps (vidéo), 60 fps (jeu/web). Définit la durée réelle de l'animation.

Interpolation

Calcul automatique des valeurs entre 2 keyframes. Contrôlé par les F-Curves dans le Graph Editor.

F-Curve

Courbe dans le Graph Editor qui définit l'évolution d'une valeur dans le temps. Editable avec des poignées Bézier.

Action

Bloc réutilisable d'animation contenant un ensemble de F-Curves. Peut être assigné, dupliqué, empilé dans le NLA.

NLA Editor

Non-Linear Animation — empile et mixe plusieurs Actions comme des clips vidéo. Essentiel pour l'export multi-states.

Les 3 transformations fondamentales

Location (G)
  • Déplacement de l'objet dans l'espace
  • Axes : X (rouge), Y (vert), Z (bleu)
  • Raccourci : G puis X/Y/Z
  • Propriété : object.location
Prompt : "déplace le cube de 2m en Z"
Rotation (R)
  • Rotation autour d'un axe
  • Modes : Euler XYZ, Quaternion
  • Raccourci : R puis X/Y/Z
  • Propriété : object.rotation_euler
Prompt : "tourne le cube de 90° en Z"
Scale (S)
  • Mise à l'échelle (uniforme ou par axe)
  • Uniform : même valeur X=Y=Z
  • Raccourci : S puis X/Y/Z
  • Propriété : object.scale
Prompt : "double la taille en X"
Espace Local vs World
  • Local : axes relatifs à l'orientation de l'objet
  • World (Global) : axes fixes de la scène
  • Raccourci : appuyer 2× sur l'axe (ex: G Z Z = Z local)
Prompt : "déplace en Z local" vs "déplace en Z world"
Pivot Point
  • 3D Cursor : pivot au curseur 3D
  • Median Point : centre de la sélection
  • Individual Origins : chaque objet tourne sur lui-même
  • Active Element : pivot sur l'élément actif
Prompt : "tourne autour du curseur 3D"

Shape Keys / Morphing

Principe
  • Les Shape Keys définissent des formes alternatives du maillage
  • Basis = forme de référence (repos)
  • Chaque key a une valeur (0.0 → 1.0)
  • Animer la valeur = morpher entre les formes
  • Propriété : object.data.shape_keys.key_blocks['Key'].value
Idéal pour : visages, déformations organiques, UI animées
Cas d'usage typiques
  • Expressions faciales (smile, blink, surprise)
  • Transformation d'un objet en un autre
  • Respiration / gonflement organique
  • Boutons UI qui changent de forme
  • Liquide qui se déforme
Combinable avec scale + rotation pour plus d'impact
# Exemples de prompts pour Shape Keys
"Crée une shape key 'Smile' qui relève les coins de la bouche, anime de 0 à 1 sur 30 frames"
"Morphe la sphère en cube via shape key, transition sur 2 secondes avec ease in-out"
"Ajoute un shape key 'Breathe' qui gonfle le torse, boucle sinusoïdale 0.0→0.6→0.0"
"Anime les shape keys 'Blink_L' et 'Blink_R' en sync : 0→1→0 sur 8 frames, toutes les 3s"

Modificateurs F-Curve

Cycles
  • Mode REPEAT : boucle identique à l'infini
  • Mode MIRROR : boucle en aller-retour
  • Paramètres : Before/After, Count
"en boucle infinie" / "ping-pong"
Noise
  • Variation aléatoire à la courbe
  • Paramètres : Scale, Strength, Phase, Depth
  • Strength faible = tremblement subtil
"avec du bruit" / "tremblement léger"
Stepped
  • Discrétise le mouvement (paliers)
  • Effet stop-motion / saccadé
  • Paramètres : Step Size, Offset
"style stop-motion" / "mouvement saccadé"
Limits
  • Clampe les valeurs min et/ou max
  • Empêche un objet de dépasser une zone
  • Combinable avec Noise ou Cycles
"limité entre 0 et 2m en Z"
Envelope
  • Module l'amplitude de la courbe
  • Points de contrôle d'intensité
  • Fade in / fade out d'animation
"commence doucement, intensifie, puis calme"
# Exemples de prompts F-Curve modifiers

Drivers & Expressions

Principe des Drivers
  • Un driver remplace l'animation manuelle par une formule
  • La propriété driven est calculée automatiquement
  • Variables : propriétés d'autres objets, frame, custom props
  • Fonctions : sin(), cos(), min(), max(), clamp()
Propriété driverée = champ violet dans Blender
Cas d'usage courants
  • Rotation pilotée par la position (roue qui tourne en avançant)
  • Scale proportionnel à la distance
  • Oscillation automatique via sin(frame / speed)
  • Visibilité conditionnelle (si Z > seuil, visible)
Combinable avec constraints pour des rigs complexes
FormuleEffetPrompt FR
sin(frame / 30) * 0.5Oscillation sinusoïdale (période ≈60f)"oscille en sinus"
var_loc_z * 2.0Scale = 2× la hauteur Z"grossit quand il monte"
clamp(var, 0.0, 1.0)Valeur limitée entre 0 et 1"jamais en dessous de 0"
noise.random() * 0.1Variation aléatoire à chaque frame"bruit aléatoire subtil"
-var_rot_zRotation inverse de la cible"tourne à l'opposé de X"
 
# Prompt driver
"Ajoute un driver sur scale.z : sin(frame / 45) * 0.3 + 1.0 (pulsation automatique sans keyframes)"

Prompting efficace

Structure d'un prompt optimal

[OBJET]  +  [ACTION]  +  [AXE/DIRECTION]  +  [VITESSE/DURÉE]  +  [EASING]  +  [BOUCLE]
Basique
"Fais tourner le cube autour de Z, 1 tour complet en 120 frames, en boucle"
Objet: cube | Action: rotation | Axe: Z | Durée: 120f | Boucle: oui
Interméd.
"Le logo flotte en Z (±0.2m, 60 frames, ease in-out) et tourne lentement en Y (1 tour / 180 frames)"
2 actions | Amplitudes précises | Timings différents | Easing spécifié
Avancé
"Pop-in du produit : scale 0→1.15→1.0 sur 25 frames (Back easing), puis lévitation Z ±0.1m en boucle, caméra dolly avant lent sur 90 frames"
3 animations séquencées | Valeurs exactes | Easing | Boucle partielle
💡 ASTUCE Plus le prompt est précis (valeurs numériques, axes, timing), plus le résultat sera conforme. Évitez les descriptions vagues comme "anime l'objet" ou "fais bouger le truc".

Anti-patterns

❌ "Anime l'objet"
Trop vague — quel type d'animation ? rotation, translation, scale ?
✅ Corrigé
"Fais tourner le cube en Z, 1 tour en 120 frames, boucle"
❌ "Fais bouger le truc"
Pas d'objet précis, pas de direction, pas de paramètres.
✅ Corrigé
"Déplace la sphère de 3m en X sur 60 frames, ease out"
❌ Oublier l'axe
"Fais tourner" — autour de quel axe ? Résultat imprévisible.
✅ Corrigé
Toujours spécifier X, Y ou Z (et local vs world si ambigu)
❌ Pas de timing
"Fais rebondir" — combien de frames ? quelle hauteur ?
✅ Corrigé
"Rebondit en Z depuis 2m, 3 rebonds, 90 frames total"
❌ Oublier le bake/export
L'animation fonctionne dans Blender mais ne s'exporte pas.
✅ Corrigé
"Bake all actions, puis exporte en GLB avec Draco"
❌ Animations > 5s pour le web
Fichiers GLB trop lourds, temps de chargement excessif.
✅ Corrigé
Viser 2-4 secondes max, boucle parfaite, compression Draco
❌ Trop d'effets simultanés
Rotation + translation + scale + morph = chaos illisible.
✅ Corrigé
Max 2-3 effets combinés, avec des rythmes différents

Export Web

▾

Pipeline d'export animation → GLB

1

Bake All Actions

  • Object → Animation → Bake Action
  • Convertit drivers/constraints en keyframes
  • Nécessaire pour l'export correct
Prompt : "bake toutes les animations"
2

NLA Editor

  • Push Down chaque Action dans le NLA
  • Gérer plusieurs clips d'animation
  • Séparer idle / hover / click states
Prompt : "push down les actions dans le NLA"
3

Export GLB

  • Format glTF 2.0 (.glb = binaire compact)
  • Apply Modifiers ✓, Bake Animation ✓
  • Sampling Rate : 1 ou 2 pour alléger
Prompt : "exporte en GLB, apply modifiers"
4

Compression Draco

  • Réduit la taille de 50-80%
  • Activer dans les options d'export GLB
  • Compression level : 6 (bon compromis)
Prompt : "active la compression Draco level 6"
# Prompt complet d'export
"Bake toutes les animations (Bake Action, Visual Keying activé),
 push down dans le NLA,
 puis exporte en GLB (.glb) avec :
   - Apply Modifiers activé
   - Bake Animation activé, sampling rate 1
   - Compression Draco level 6
   - Fichier : produit_anime.glb"

Checklist finale

✓ Spécifier l'objet par son nom exact
✓ Vérifier le frame range (début et fin)
✓ Préciser le type d'animation (rotation, translation, scale…)
✓ S'assurer que frame 1 = frame N+1 pour les boucles
✓ Indiquer l'axe (X, Y, Z) et l'espace (local/world)
✓ Bake les animations avant export
✓ Donner des valeurs numériques (degrés, mètres, frames)
✓ Utiliser le NLA pour les animations multi-states
✓ Spécifier l'easing (Bezier, Ease In-Out, Back, Bounce…)
✓ Exporter en GLB (pas glTF séparé) pour le web
✓ Préciser si l'animation doit boucler
✓ Activer la compression Draco
✓ Limiter à 2-3 effets combinés maximum
✓ Tester la boucle dans un viewer GLB
✓ Viser 2-4 secondes pour les animations web
✓ Vérifier la taille du fichier (< 1MB idéal)

WORKFLOW RÉSUMÉ

  1. Décrire l'animation avec précision (objet + action + axe + timing + easing + boucle)
  2. Claude Code génère le script Python Blender
  3. Vérifier le résultat dans le viewport (jouer la timeline)
  4. Ajuster si nécessaire ("rends le plus lent", "ajoute un ease out", …)
  5. Bake → NLA → Export GLB Draco → Intégrer dans le site web
